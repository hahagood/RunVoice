# RunVoice UI 审查记录

日期: 2026-03-30

审查方式:
- 结合 Compose 源码阅读
- 基于真机页面截图复核
- 覆盖页面: 主跑步页、心率带设置页、About 页

## 总结

当前 UI 的主要问题不是配色，而是信息层级、空间分配和状态表达不够清晰。

优先级最高的优化方向:
1. 重构主跑步页布局，减少“竖向堆卡片”的拥挤感。
2. 修正主按钮语义，让“暂停”和“结束”行为与文案一致。
3. 给心率带设置页补齐空状态和连接状态的表达。
4. 把 About 页从“说明书”改成更像产品介绍页的结构。
5. 给所有页面统一补上系统栏安全区适配。

## 主跑步页

### 主要问题

1. 四张数据卡片权重几乎完全一样，导致页面没有明确视觉中心。
   时间、心率、配速、距离都用了同一类大卡片，用户视线没有主落点。

2. 页面纵向空间利用效率偏低。
   内容不算多，但首屏已经接近被填满，整体观感偏“堆盒子”。

3. 空状态的数据样式太抢眼。
   没有心率或配速时，依然用高饱和红色和黄色展示占位值，看起来像异常告警，而不是“暂无数据”。

4. 顶部头部区域过轻。
   左侧 `RunVoice` 和右侧心率带状态更像两个零散链接，不像完整 header。

5. 节拍器控件的可操作感不足。
   当前更像一行文本和箭头，而不是一个明确的工具控制条。

6. 按钮文案和行为不一致。
   运行中主按钮现在触发的是暂停，但文案写的是“停止跑步”，容易误导。

### 建议改法

1. 改成 `主指标 + 次指标 + 工具条 + 主按钮` 的结构。
2. 把时间和距离设为一级信息，心率和配速降为二级卡片。
3. 把空状态颜色降到低对比度灰色。
4. 顶部做成真正的头部: 左边品牌，右边设备状态徽标。
5. 把“停止跑步”改成“暂停跑步”，暂停后再出现“继续”和“结束”。

## 心率带设置页

### 主要问题

1. 当前状态下页面下半部分几乎是空的，完成度偏低。
   只有“已保存设备”卡片和一个扫描按钮，之后是大面积空白。

2. “已保存设备”卡片的主次信息不清晰。
   MAC 地址最显眼，但用户真正关心的是“当前是否已连接”和“下一步该做什么”。

3. 缺少空状态引导。
   没有扫描时、扫描不到设备时、未连接时，页面都没有给出解释性文案。

4. 主按钮文案偏工程化。
   “扫描心率设备”功能上没错，但产品表达更自然的写法应该是“搜索设备”。

5. “断开并清除”视觉权重偏高。
   这是次级危险操作，不应该与主操作抢注意力。

6. 顶部栏布局是手工凑平衡，不够自然。
   标题虽然居中，但结构不够稳定，后续扩展也不方便。

### 建议改法

1. 页面拆成三段: 顶部栏、当前设备状态卡、设备列表或空状态区。
2. 当前设备卡突出设备名称和连接状态，MAC 地址降成次级文本。
3. 增加空状态文案:
   - 还未搜索设备
   - 正在搜索
   - 未发现设备
   - 已连接
4. 主按钮文案改成“搜索设备”，扫描中改成“停止搜索”。
5. 将“断开并清除”降级成弱操作按钮或文字按钮。

## About 页

### 主要问题

1. 现在更像一份说明书，而不是产品内页面。
   真机上看是连续的大段文字，阅读压力偏高。

2. 标题下缺少一句简洁副标题。
   页面没有先告诉用户“这是什么”“适合什么场景”，而是直接进入长文。

3. 各 section 只有文字样式区分，没有版式分层。
   看起来偏平，信息块之间的节奏不够明确。

4. “免责声明”与前两个 section 视觉权重相近，但它本应更次级。

5. 顶部只有轻量返回按钮，不像正式页面 header。

6. 文案整体偏长，移动端扫读效率一般。

### 建议改法

1. 把 About 页改成“产品介绍页”而不是纯说明页。
2. 标题下增加一句副标题，例如:
   `语音驱动的轻量跑步记录工具`
3. 把功能简介和使用说明改成卡片式模块。
4. 将功能点压缩成更短的 bullet 列表。
5. 免责声明单独做成弱化卡片、折叠区，或放在页面底部。
6. 如果这个页面继续承担首次引导功能，需要补一个明确主按钮，例如“开始使用”。

## 跨页面问题

1. 尚未统一处理系统栏安全区。
   当前页面主要依赖固定 `padding(24.dp)`，在全面屏设备上会有贴边感。

2. 三个页面的视觉语言方向一致，但组件层级不够统一。
   例如 header、状态卡、主按钮、次按钮的权重体系还没有真正建立起来。

3. 空状态设计明显不足。
   当前多个页面更像“有数据时的界面”，对“暂无数据”“未连接”“未开始”的状态照顾不够。

## 推荐实施顺序

1. 先改主跑步页。
   这是使用频率最高的页面，收益最大。

2. 再改心率带设置页。
   结构相对简单，能快速拉升整体完成度。

3. 最后重做 About 页。
   它更偏信息设计，可以在主流程稳定后统一处理。
