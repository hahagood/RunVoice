# RunVoice Roadmap

## 当前阶段

1. 耳机单击触发即时播报
   目标: 跑步过程中不看表，只通过耳机按键就能请求一次当前数据播报。
   状态: 已恢复 MediaSession 媒体键链路，并完成真机验证。

2. 结束跑步确认页
   目标: 避免在暂停后误触“结束本次”直接结束跑步。
   状态: 已实现页内确认流程，可选择保存数据、放弃本次或继续跑步。

## 下一阶段

1. 跑步数据保存
   目标: 保存每次跑步的基础摘要数据。
   当前说明: 现阶段已支持保留或丢弃 GPS 轨迹 CSV，但还没有正式的跑步摘要历史表。
   第一版建议字段:
   - 开始时间
   - 跑步时长
   - 距离
   - 平均配速
   - 当前/平均/最大心率

2. 历史记录与汇总页
   目标: 能查看最近跑步记录，并给出最基础的汇总信息。
   第一版建议:
   - 最近跑步列表
   - 周跑量 / 月跑量
   - 最长距离
   - 最快配速
   - 最大心率记录

## 更后面的阶段

1. 即时训练分析
   目标: 在跑步过程中给出更有意义的训练状态判断。
   候选能力:
   - 心率区间（Zone）判断
   - 有氧 / 无氧提示
   - 心率过高提醒
   - 节奏异常提醒

2. 历史分析能力
   目标: 从多次跑步数据中看趋势，而不是只看单次结果。
   候选能力:
   - 心率趋势
   - 配速稳定性
   - 恢复跑 / 训练跑分类
   - 周期训练对比

## 实施原则

1. 优先做“自用时最直接有价值”的功能。
2. 优先保证播报、心率和即时读数可靠，再扩展分析。
3. 先做可用的最小版本，不急着一次补全所有训练系统功能。
